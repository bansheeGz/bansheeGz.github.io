BGDatabase v.1.9.1 The performance upgrade for codegen classes

08/2025

Description

In BGDatabase v.1.9.1 we improved performance for generated classes. Read this guide for more details.

  1. Backup your database file(s) just in case before upgrading
  2. After the upgrade, ensure the BGDatabase package build number is 2025.08.10 (it's a date, August 10, 2025). If it's lower, please update the BGDatabase package from the Package Manager.

Minor API changes

Some APIs have been changed. We hope that very few people (if any) are affected by these changes. Fortunately, none of our plugins are affected (some of them need to be updated to work properly with new features). However, if you are using custom scripting with our API, you may encounter some breaking changes. Hereâ€™s how to fix them.

Prior to version 1.2, each field and table had a single corresponding class, such as BGFieldInt for integer fields, BGMetaRow for "row" tables etc. Now, some implementing classes may differ, and we have switched to using interfaces for these classes, for example, the interface for the integer fields is BGFieldIntContract and the interface for the "row" tables is BGMetaRowContract etc. A few method signatures have been updated to work with interfaces instead of classes. You can cast these interfaces to their corresponding base classes: BGField is the base class for all fields, and BGMetaEntity is the base class for all tables.

If you encounter a compilation error related to the API change that you cannot resolve, please revert the database to version 1.8.19 without saving the database with the new version and contact us.
