Code Generation

Code Generation is a key element for simplifying access to database. There are 2 main types of code generation, lets look at them.
  1. Extension classes
  2. MonoBehaviour Classes

Extension classes

These classes are inherited from basic BGEntity. These classes can not be added to GameObject, cause they are not Unity's MonoBehaviour. When this CodeGen is enabled, database populated not with basic BGEntity objects, but with these extension classes. So when you access database like this var entity=meta[0]; entity variable has this extension type, with all fields generated as properties.

This CodeGen is embedded to database via Database addon, called CodeGeneration. After enabling this addon you will be able to generate these classes.

Here is an example. Let's say you have a table called MyTable which has one field, called intField of type int. So the generated class will look like this 

public partial class BGE_MyTable : BGEntity
{
    public System.Int32 f_intField
	{
		get
		{
			return Get<System.Int32>("intField");
		}
		set
		{
			Set("intField", value);
		}
	}
}

As you can see all boilerplate code will be generated, so you can simply use entity.f_intField to get or set value of type int

Additionally, you will get all main table(meta) methods overridden as static methods of that particular class, like this: 

public static BGE_MyTable FindEntity(Predicate<BGE_MyTable> filter)
{
}
public static List<BGE_MyTable> FindEntities(Predicate<BGE_MyTable> filter)
{
}
public static void ForEachEntity(Action<BGE_MyTable> action, Predicate<BGE_MyTable> filter=null)
{
}
public static BGE_MyTable GetEntity(BGId entityId)
{
}
public static BGE_MyTable GetEntity(int index)
{
}
public static BGE_MyTable NewEntity()
{
}

so instead of doing this, for example: 

    var repo = BGRepo.I;
    var meta = repo["MyTable"];
    var entity = meta[new BGId("HThOdRCxlEGU1aefSG+Nbw")];
you can do this: 
    var entity = BGE_MyTable.GetEntity(new BGId("HThOdRCxlEGU1aefSG+Nbw"));

using generated BGE_MyTable class.

MonoBehaviour Classes

Those classes are inherited from BGEntityGo, which in turn is inherited from Unity's MonoBehaviour. So these classes are Unity standard components and can be added to any GameObject in the Inspector. After that you can chose which row should be connected. So these classes are designed to be added to GameObject in the Inspector

This CodeGen is embedded to database via Database addon, called CodeGeneration. After enabling this addon you will be able to generate these classes.

This classes is not embedded to database and can be generated on Settings tab in the BGDatabase window
