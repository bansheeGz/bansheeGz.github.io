Current state of multithreading support

  1. Multithreading is not currently supported. Database is not thread-safe. It's meant to be called from main thread only
  2. We are looking for opportunities to add multithreading and integration at some level with Unity's Jobs and ECS

How to use database with Unity's ECS and Jobs

  1. Make sure you call database only from main thread (e.g. from MonoBehaviour classes)
  2. Create Unity ECS entities out of database information from MonoBehaviour class
  3. If you need some of these Unity ECS entities to be persistent, (e.g. they needs to be saved/loaded), store them back to database the moment they needs to be persisted, e.g, when the game needs to be saved (see example below)
  4. If by some reason you need to call database from multithreaded environment, use the class below. Make sure you call at sparingly, at some relatively rare condition, otherwise it can become a bottleneck.
