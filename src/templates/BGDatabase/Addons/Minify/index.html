#set page=Minify add-on

Notice

This is work-in-process, expected to be release in version 1.8.18

Description

"Minify" add-on removes selected tables or fields from the database file included in the build. This add-on activates during the player build start event, removing selected tables and fields, and subsequently restores the full database upon the build completion event.

Important note

If an unexpected event occurs during the build process, such as a crash or power outage, you may end up with an incomplete/minified database. In this case, you need to check if database is minified and restore it from either local backup folder or from temp folder, created by minify addon. You can find temp folder location under "Misc" tab

Detecting accidental access to removed tables or fields

When you are running your game in Unity Editor, you have access to the full database, so if you accidentally access table/fields, marked for exclusion, there will be no error. However, when you build and run your game outside of Unity Editor, an exception will be thrown, because tables or fields were removed from the database. Unfortunately, there is no way to detect such accidental access to excluded tables/fields if you use basic API, but in case of code generation, we can take some actions to detect such invalid access. The following actions available:

Name Description
Skip Disabled no code will be generated for excluded tables/fields, so you won't be able to access them via generated classes.
Surround with #if UNITY_EDITOR code for excluded tables/fields will be surrounded with UNITY_EDITOR directive, so it works well inside Unity Editor, but if you accidentally access this code from runtime assembly, compilation error occurs during player build process. This option is effective for extension classes and MonoBehaviour classes only.
Throw exception exception is thrown if you access excluded table or field during playmode is on
Log warning warning is printed to the console if you access excluded table or field during playmode is on
Log debug debug message is printed to the console if you access excluded table or field during playmode is on
