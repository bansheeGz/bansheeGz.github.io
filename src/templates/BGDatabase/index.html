Fast, InMemory, pure c# (without SQL), Extendable Database for your Unity 3D in-game data with export/import to Excel and Google Sheets, which works like Unity does by reverting runtime changes
Version:
1.6.x (change log)

Support thread

Features:

  1. Fast, In Memory, pure C# ( without SQL) database fully integrated into Unity 3D
  2. Selective import/export from/to Excel/OpenOffice and Google Sheets (some restrictions apply)
  3. Integration with visual scripting tools: Unity Visual Scripting (Bolt), Playmaker, Flow Canvas, Node Canvas, Behavior Designer, UNode, Game Creator.
  4. Powerful Code Generation
  5. Selective Save/Load during game session
  6. Over 50 field types, including relations and nested tables.
    Support for loading Unity assets from Resources, AssetBundles and Addressables system
  7. Hook up any GameObject to table row by adding a component
  8. Runtime data editor
  9. Ability to create your own custom field.
  10. Merge tool for reviewing and selectively applying the differences between 2 databases
  11. Live update from Google sheets
  12. Support for modding with Modding addon
  13. Auto-import data from Excel file in Editor and at runtime (details)
  14. No runtime dependencies, dll size ~300Kb (0.3Mb).
  15. Downloadable plug-ins and example projects
  16. Lazy loading
  17. Flexible event system
  18. Multithreading is supported with multithreading addon. Some restrictions apply. Read more here.
  19. BGDatabase is a client-side database, which runs inside Unity only. It can not be used in any server environment
