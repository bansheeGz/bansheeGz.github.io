General information

Database consists of 4 main substances:
  1. Table (called meta)
  2. Table field
  3. Table row (called entity)
  4. Database addon
todo image Every table, field and entity has unique id (based on Guid generation) and name (table and field has unique name, 32 char max)

Accessing the database

Once you setup and created a new database, you can access it in 2 main ways:
  1. Through code
  2. You can hook up any GameObject to database row by adding BGEntityGo component to it and choosing a row.

Accessing database by code without code generation

BGDatabase has very easy, slim and minimalistic API. However using CodeGeneration this API can be even easier (we strongly recommend to use it). Learn more about Code Generation here

Here is how you can access database without code generation: 

         //get access to database
        var repo = BGRepo.I;

        //====================      WORKING WITH TABLES (called metas)
        //get access to table by name(called meta)
        var meta = repo["MyTable"];

        //iterate over all tables, whose name start with "My"
        repo.ForEachMeta(m =>
        {
            //do something
        }, m => m.Name.StartsWith("My"));

        //number of tables
        var metaCount = repo.CountMeta;

        //check if table exists
        var isMetaExists = repo.HasMeta("MyTable");

        //some methods are ommitted!

        //====================      WORKING WITH TABLE ROWS (called entities)
        //iterate over all rows, whose name start with "My"
        meta.ForEachEntity(e =>
        {
            //do something
        }, e => e.Name.StartsWith("My"));

        //number of entities
        var entityCount = meta.CountEntities;

        //get entity by id
        var entity = meta[new BGId("some-id-here")];

        //find first entity with name starting with "My"
        entity = meta.FindEntity(e => e.Name.StartsWith("My"));

        //get field value from entity (int should be substituted with actual field type)
        var value = entity.Get<int>("someIntField");

        //set field value to entity
        entity.Set("someIntField", value);

        //some methods are ommitted!

        //====================      WORKING WITH TABLE FIELDS
        //is field exists
        var isFieldExists = meta.HasField("MyField");

        //get field
        var field = meta.GetField("MyField");

        //get field with type int
        var intField = meta.GetField<int>("MyField");

        //some methods are ommitted!

Accessing database by adding BGEntityGo component

You can easily hook up any GameObject to a database row by adding BGEntityGo component to it. Just add the component, select table (meta) and then select a row (entity) todo image After that all fields will be immediately available for reading and writing.
